Key Files Explained:
====================

Backend:
--------
- server.ts              : Express server with routes and middleware
- auth.ts (middleware)   : Verifies JWT tokens from Supabase
- flashcards.ts (route)  : API endpoints for flashcard operations
- supabase.ts (config)   : Supabase client initialization

Frontend:
---------
- App.tsx                : Main app with routing configuration
- AuthContext.tsx        : Global authentication state & methods
- ProtectedRoute.tsx     : Wraps routes requiring authentication
- Dashboard.tsx          : Layout with header, sidebar, and content
- Flashcards.tsx         : Book display and generation interface
- FlashcardViewer.tsx    : Card flip and navigation interface

Configuration:
--------------
- .env (backend)         : Supabase keys, port, CORS settings
- .env (frontend)        : Supabase keys, API URL
- package.json           : Dependencies and scripts
- tsconfig.json          : TypeScript compiler options
- tailwind.config.js     : Tailwind CSS theme customization

Data:
-----
- flashcards.json        : 100 pre-generated flashcards
- fortyHadiths.pdf       : Source book (2.2 MB)
- chroma_db_nccn/        : Vector embeddings for AI features

Important Notes:
================

1. The backend reads flashcards from JSON files (no AI generation in MVP)
2. User state is stored in Supabase database
3. Authentication is handled by Supabase
4. Frontend uses React Router for navigation
5. Tailwind CSS for styling
6. TypeScript throughout for type safety
7. Environment variables required for both backend and frontend

File Counts:
============
- Backend TypeScript files: 6

Getting Started:
===============
1. Read docs/QUICK_START.md
2. Set up Supabase (docs/SUPABASE_SETUP.md)
3. Configure environment variables
4. Run: cd backend/api && npm run dev
5. Run: cd frontend && npm run dev
6. Open: http://localhost:5173
